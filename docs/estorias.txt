# Estórias do usuário

Feature: Autenticação de Usuários

Como um usuário Professor ou Estudante,
Para poder acessar o sistema,
Eu quero poder me autenticar.

Feature: Cadastro de Questões

Como um Professor,
Para cadastrar questões,
Eu quero poder cadastrar questões do tipo múltipla escolha, verdadeiro ou falso e do tipo valor numérico.

Feature: Criação de Exames

Como um Professor,
Para criar exames,
Eu quero poder cadastrar um exame contendo um número arbitrário de questões, com uma data e horário para abertura e encerramento e atribuir um valor a cada questão presente no exame.

Feature: Relatórios de Exames

Como um Professor,
Para acompanhar o desempenho dos estudantes,
Eu quero poder visualizar um relatório com as respostas dos estudantes e as notas obtidas por cada estudante.

Feature: Realização de Exames

Como um Estudante,
Para realizar um exame,
Eu quero poder responder às questões definidas para este exame e concluí-lo. Um exame concluído não pode ser editado.
Eu quero poder visualizar a nota e as questões acertadas e erradas de um exame realizado e já encerrado.

Feature: Listagem de Dados de Exames

Como um Estudante,
Para obter informações sobre os exames,
Eu quero poder listar e visualizar dados de exames.

Feature: Controle de Realização de Exames

Como um Estudante,
Para realizar um exame dentro do prazo correto,
Eu quero poder realizar o exame apenas se a data presente estiver entre o momento de abertura e encerramento e que seja automaticamente concluído no momento de encerramento.

Feature: Responsividade com diferentes dispositivos (computadores, tablets e smartphones).
Feature: Utilizar um banco de dados, sendo possível em ambiente de testes utilizar o SQLite.
Feature: Seguir a arquitetura MVC
Feature: Seguir princípios DRY quanto aos templates, controladores e Models. 
Feature: Rotas devem seguir o padrão RestFul.
